### Challenges and Friction

- The same CommandApi controller area accumulated several debugging algorithms. This kept reuse high, but it also concentrated risk around JSON reconstruction, diffing, truncation behavior, and DAPR trust-boundary assumptions.
- Multiple tools reconstruct state from event payload JSON rather than a typed aggregate state model. That made the tooling feasible and domain-agnostic, but it inherits limitations around field deletion, nested object removals, array semantics, and domains whose event payloads are not state-reflecting.
- Several tools rely on `GetEventsAsync(0)` or tail scanning. The story records repeatedly identify full-stream reads as accepted technical debt for large streams.
- URL mode precedence in `StreamDetail.razor` became progressively more complex: sandbox > trace > step > bisect > blame > diff. The final behavior is useful, but this area needs regression protection whenever another stream tool is added.
- Story 20.5 intentionally introduced new trace controllers because correlation traces are cross-stream concerns. That was the right exception, but it broke the simpler "add everything to streams" habit from stories 20.1-20.4.
- Live proof is thinner than unit proof. Story records report strong Tier 1 coverage, and this retrospective successfully started the Aspire AppHost, but no manual browser/debugging workflow sweep was performed during the retro.

### Breakthrough Patterns

- StreamDetail became an investigation console: state, diff, blame, bisect, step-through, sandbox, and trace all share the same aggregate context.
- Diagnostics stayed read-only or explicitly dry-run. The sandbox invokes the domain service but does not persist events, status records, or archive entries.
- Correlation IDs became navigation objects, not inert text. This directly advances FR72 and the admin UI's observability-deep-link strategy.
- The epic used graceful degradation where system truth can expire or be unavailable: missing command status, missing projection status, absent external trace URL, unavailable domain service, and too-large field sets all return structured outcomes.
- Review findings improved the diagnostic contract: timeout propagation, bad-request behavior, cancellation handling, URL deep-link precedence, and exact-vs-substring watch-field semantics all got sharpened.

### Key Takeaways

1. The strongest debugging tools answer a chain of questions, not a single query: what changed, when did it diverge, what would happen, and where did the correlation go.
2. Domain-agnostic JSON reconstruction enabled rapid tool delivery, but its limitations now need an explicit hardening plan.
3. Deep-link precedence is now product behavior. It deserves tests whenever new stream modes are added.
4. Dry-run and read-only semantics are valuable because they let operators investigate without mutating event-store truth.
5. Unit evidence is strong, but advanced debugging tools need live scenario evidence with real streams, correlations, projections, and domain-service invocation.
6. Deferred review findings are accumulating around the same core seams: event enumeration, JSON merge/diff fidelity, cancellation, and trust-boundary enforcement.
