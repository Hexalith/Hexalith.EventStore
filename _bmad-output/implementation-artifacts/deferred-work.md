## Deferred from: code review of post-epic-11-r11a1-checkpoint-tracked-projection-delivery (2026-05-01)

- **Checkpoint > actor's `CurrentSequence` permanently silences projections** — If the persisted `ProjectionCheckpoint.LastDeliveredSequence` exceeds the aggregate's `CurrentSequence` (restored state-store backup, key drift), `AggregateActor.GetEventsAsync` returns `[]` for the higher-than-current `fromSequence`, the orchestrator hits `Log.NoEventsFound` and returns, and the tracker never advances. Future events are silently skipped. Add drift detection in a follow-up story (e.g., warn when `events.Length == 0 && lastDeliveredSequence > 0`, or compare checkpoint to actor metadata). `src/Hexalith.EventStore.Server/Projections/ProjectionUpdateOrchestrator.cs:80-83`, `src/Hexalith.EventStore.Server/Actors/AggregateActor.cs:592-594`.
- **Concurrent fire-and-forget projection triggers can regress projection state** — `EventPublisher` spawns `Task.Run` per publish without per-aggregate serialization. Two concurrent triggers can read checkpoint=N independently, fetch overlapping event ranges, and `IProjectionWriteActor.UpdateProjectionAsync` calls can land out of order so a stale projection state sits behind a high checkpoint until a new event arrives. The design doc honestly concedes "duplicate delivery can still occur under concurrent fire-and-forget triggers" but does not address state-regression. Per-aggregate `Channel<T>` or `SemaphoreSlim` is the right fix; track in R11-A2 or successor. `src/Hexalith.EventStore.Server/Events/EventPublisher.cs:135-143`.
- **Outer orchestrator catch absorbs `OperationCanceledException` from checkpoint read** — `ProjectionUpdateOrchestrator.UpdateProjectionAsync`'s outer `catch (Exception ex)` has no OCE filter (line 151). A future caller passing a real `CancellationToken` would have OCE logged as `ProjectionUpdateFailed` instead of cooperating with cancellation. Pre-existing pattern; not caused by R11-A1. EventPublisher passes `CancellationToken.None` so production is unaffected today. `src/Hexalith.EventStore.Server/Projections/ProjectionUpdateOrchestrator.cs:151`.
- **`MaxEtagRetries = 3` not exposed via `ProjectionOptions`** — Hardcoded in `ProjectionCheckpointTracker`. Story Dev Notes explicitly defer the retry count to dev-story discretion provided the bounded/non-regression contract is preserved. Add a `CheckpointMaxEtagRetries` option in a follow-up if hot-spot aggregates routinely hit `CheckpointSaveExhausted`. `src/Hexalith.EventStore.Server/Projections/ProjectionCheckpointTracker.cs:165`.
- **`SaveDeliveredSequenceAsync` rewrites identical state when caller passes a sequence ≤ existing checkpoint** — Costs a state-store round-trip and updates `UpdatedAt` for no observable change. Short-circuiting `if (existing?.LastDeliveredSequence >= deliveredSequence) return true;` is a future micro-optimization. Perf NIT only. `src/Hexalith.EventStore.Server/Projections/ProjectionCheckpointTracker.cs:198-218`.
- **`AggregateActor.GetEventsAsync` `(int)(fromSequence + 1)` `checked` cast can overflow at `int.MaxValue` checkpoint** — Pre-existing actor cast at `AggregateActor.cs:596`. Previously unreachable because the orchestrator always passed `0`; now reachable through a persisted long checkpoint. The orchestrator's outer catch silently swallows the resulting `OverflowException`. Pre-existing actor bug, not caused by R11-A1.
