## 2. What Went Well

1. **Shell-first foundation (15-1)** stabilized middleware, auth, CSS/JS interop, responsive framework on day one. Zero rework across 13 downstream stories.
2. **Reusable service layer** тАФ `AdminStreamApiClient` pattern (15-2) replicated exactly in `AdminProjectionApiClient` (15-5) and `AdminTypeCatalogApiClient` (15-6). No ad-hoc HTTP calls sprinkled through components.
3. **Dual-layer refresh** (30s polling + SignalR enhancement) proved resilient. Polling guarantees freshness; SignalR reduces latency when available. Graceful degradation standardized across 15-2, 15-3, 15-5, 15-6, 15-9.
4. **Master-detail responsive pattern** locked by 15-3 (`FluentSplitter` тЙе 1280px, conditional Razor rendering below). Reused unchanged in 15-4 / 15-5 / 15-6 / 15-9 / 15-12.
5. **StatusBadge factory** тАФ one component, many `StatusDisplayConfig.From*()` factories (CommandStatus, StreamStatus, TimelineEntryType, HealthStatus, ProjectionStatusType, EventTypeInfo). Scaled to six status enums without component duplication.
6. **Deep-link audit (15-8)** confirmed URL-state round-trips across Streams, Projections, TypeCatalog, Health, Events. Zero URL-state surprises at epic close.
7. **BMAD 3-layer code review** (Blind Hunter + Edge Case Hunter + Acceptance Auditor) caught concurrency (`Interlocked` race on `_isRefreshing`), `JSDisconnectedException` guards, fire-and-forget exception swallowing, stale-data indicator gaps. Average 3тАУ7 patches per story, all tactical.

---

## 3. What Struggled

1. **DAPR query cursor gaps** тАФ server exposes `fromSequence` / `toSequence` / `count`, no `offset` / `limit`. Client-side `Skip/Take` over a `count=1000` fetch works up to ~10K items; 15-2, 15-9, 15-12 all carry placeholder comments flagging the scale boundary.
2. **DAPR service-invocation error swallowing (15-10)** тАФ 8 methods in `DaprStreamQueryService` silently returned empty results, showing "0 items" instead of failure. Cost 15-2 / 15-3 / 15-9 repeated "why is my page empty" debug cycles before 15-10 audited and rethrew.
3. **Timestamp-based state lookups (15-4)** тАФ server has no `timestamp` param. Client has to fetch timeline pages to find nearest entry, then call state-at-position. Correct but indirect.
4. **Fire-and-forget exception swallowing** тАФ `_ = InvokeAsync(async () => { ... })` caught-and-dropped in 15-2, 15-3, 15-5, 15-6. Standardized by 15-5: try/catch with `ObjectDisposedException` guard, log other exceptions.
5. **Parallel timeline fetching race (15-12)** тАФ N+1 pattern (1 streams call + up to 50 timeline calls) wrote to a shared `List<EventRow>` from concurrent tasks. Code review caught; fixed with `ConcurrentBag` + `SelectMany` post-merge.
6. **Claim-type drift (15-10)** тАФ dev-mode JWT generator used `eventstore:admin:role` (colons) while `AdminClaimTypes` used `eventstore:admin-role` (hyphen). Silent auth failure for three stories until audit. One-character fix; symptom of distributed constants.
7. **Timeline endpoint missing (15-12a)** тАФ discovered during 15-12 code review that the timeline API returned null for non-existent sequences instead of 404, and one spec-listed endpoint wasn't implemented at all. Forced a spin-off story mid-epic.

## 4. Recurring Review Feedback Themes

- **Stale vs. empty indistinguishable** тАФ standardized IssueBanner + "(stale)" label with amber background from 15-3 onward.
- **`ConfigureAwait(false)` misapplied in Blazor Server** тАФ unsafe in component lifecycle methods. SM review corrected across 15-7.
- **JSDisconnectedException in prerender phase** тАФ wrap JS interop (`ViewportService`) in try/catch with fallback wide layout. Template by 15-3.
- **Pagination state loss on auto-refresh** тАФ user lands on page 3, refresh invalidates pagination, silently jumps to page 1. Documented as known limitation in 15-9 / 15-12.
- **Deep-link URL normalization** drift тАФ each page parsed params slightly differently until 15-8 standardized `ParseUrl()` / `UpdateUrl(replace: true)` pattern.

## 8. Key Lessons (Actionable)

1. **Shell + service layer in stories 1тАУ2, nothing else** тАФ the payoff compounds across every later story.
2. **Code-review patching is the cadence, not the exception** тАФ plan for 3тАУ7 patches per story, not zero.
3. **Deep-link audit at ~50% completion**, not at epic close. If Epic 15 had deferred the 15-8 audit to a final step, drift across 7+ pages would have been costly.
4. **Centralize auth constants in a shared package** тАФ `AdminClaimTypes` in `Admin.Abstractions`, consumed by both server and UI. Eliminates claim-type drift entirely.
5. **Document scale boundaries in dev notes** (e.g., "pagination OK below 10K items") so the next epic owner inherits the decision.
6. **Formalize a manual AppHost smoke-test gate** at end of any story touching auth, DAPR invocation, or cross-service wiring тАФ unit tests won't catch silent fallthrough.
7. **JS interop via single namespaced module** (`hexalithAdmin`) proved robust тАФ keep the pattern; consider TypeScript if interop surface grows.
