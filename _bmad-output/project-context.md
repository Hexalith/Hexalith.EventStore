## Critical Implementation Rules

### C# Language-Specific Rules

- **Contracts are `sealed record` with primary constructors** + XML `<param>` docs per parameter (see `Contracts/Replay/ReplayEventEnvelope.cs`)
- **`ConfigureAwait(false)` on every awaited call** ŌĆö CA2007 is enforced (190+ sites). Missing it breaks the build in stricter projects (it's the known cause of the `Server.Tests` build failure)
- **NO file copyright headers** ŌĆö only 3 of 721 files have one. Do **not** add MIT/ITANEO headers (this differs from sibling `Hexalith.Commons`)
- **XML docs are NOT generated by default** ŌĆö `GenerateDocumentationFile` is true *only* when `ApiReferenceBuild=true` on packable projects. Don't assume CS1591 enforcement like other Hexalith repos
- File-scoped namespaces, Allman braces, `using` outside namespace, System directives sorted first
- `_camelCase` private fields, `I`-prefixed interfaces, `Async` suffix on async methods (enforced as `warning` naming rules)
- Nullable enabled ŌĆö validate at boundaries with `ArgumentNullException.ThrowIfNull` / `ThrowIfNullOrWhiteSpace`

### Framework-Specific Rules (DAPR / Aspire / MediatR)

- **MediatR pipeline order is fixed and intentional: `Authorization ŌåÆ Logging ŌåÆ Validation`** (registered in `src/Hexalith.EventStore/Extensions/ServiceCollectionExtensions.cs`). Authorization runs first so denied requests never hit the logging "pipeline entry" line. Do not reorder `AddOpenBehavior` calls
- **Event-sourcing aggregates are pure functions:** `Handle(Command, State?) ŌåÆ DomainResult` + `Apply(Event) ŌåÆ State`. No direct state mutation; state is rebuilt by replaying events. Handle/Apply are discovered by reflection convention ŌĆö no manual registration
- **DAPR abstracts state/pubsub/config** via sidecars; access-control in `DaprComponents/accesscontrol.yaml` is **deny-by-default** (slim-mode without mTLS yields 403 on service-to-service calls)
- **Multi-tenancy is at the contract level:** identity = Domain + AggregateId + TenantId
- **AppHost changes require restarting `aspire run`** ŌĆö the app model is built at startup
- `Hexalith.Tenants` source path is resolved by 3-layout logic in root `Directory.Build.props` ŌĆö don't hardcode it

### Identity Rules (ULID, not GUID)

- **System identifiers are ULIDs.** Generate with `UniqueIdHelper.GenerateSortableUniqueStringId()`
- **`Guid.TryParse` on `messageId`/`correlationId`/`aggregateId`/`causationId` is FORBIDDEN** ŌĆö use `Ulid.TryParse` (or accept any non-whitespace string per `AggregateIdentity` rules). ULID and GUID share a 36-char shape only by coincidence (Epic 2 retro R2-A7; enforced by `ProtectedIdentifierGuidParserAuditTests`)

### Testing Rules

- **xUnit v3** + **Shouldly** assertions (`ShouldBe`, `ShouldThrow`) ŌĆö never raw `Assert.*`. NSubstitute for mocks
- **Run test projects individually** ŌĆö never solution-level `dotnet test`. Use `.slnx` for restore/build only
- **Test tiers:** Tier 1 (Contracts, Client, Sample, Testing, SignalR ŌĆö run in CI), Tier 2 (Server.Tests), Tier 3 (IntegrationTests ŌĆö needs Docker + Aspire)
- **`Hexalith.EventStore.Server.Tests` has a known build failure** (CA2007-as-error) ŌĆö excluded from baseline; don't treat its red as a regression you caused
- **Integration tests MUST assert state-store end-state** (Redis key contents, persisted CloudEvent body) ŌĆö not just 202/mock call counts. "Returned 202" is a smoke test, not an integration test (Epic 2 retro R2-A6)
- All configured tests must pass before a story is complete

### Code Quality & Style Rules

- **`.slnx` only** ŌĆö never create or use `.sln` files
- `.editorconfig` sets `CA1062`/`CA1822`/`CA2007` to **warning** at solution level, but `TreatWarningsAsErrors=true` promotes them to build-breakers; `CA1014` is disabled
- Containers via **.NET SDK container support ŌĆö no Dockerfiles.** Opt in per project with `<EnableContainer>true</EnableContainer>` + `<ContainerRepository>`; defaults (alpine base, `registry.hexalith.com`, non-root `app`, port 8080, OCI labels) live in `Directory.Build.targets`
- **Submodules: root-level only** (`Hexalith.Tenants`, `Hexalith.AI.Tools`, `Hexalith.Commons`). Never recurse into / initialize nested submodules; deinit if accidentally pulled
- **Never modify submodule files without explicit approval** ŌĆö they're shared across Hexalith repos

### Development Workflow Rules

- **Conventional Commits required** (semantic-release drives versioning): `feat` ŌåÆ minor, `fix` ŌåÆ patch, `feat!`/`BREAKING CHANGE:` ŌåÆ major; `docs`/`refactor`/`test`/`chore`/`perf` ŌåÆ no bump. Don't use `feat` for refactors (false minor bump + NuGet publish)
- **Branches:** `feat/ŌĆ”`, `fix/ŌĆ”`, `docs/ŌĆ”`. No direct commits to `main`
- **Senior code review is a mandatory pipeline stage** ŌĆö budget for review-found rework (Epic 2: 5/5 stories patched). Verify CRITICAL findings before accepting (false-positive CRITICALs are expensive ŌĆö R1-A8 verification-command rule)
- Release on merge to main: test ŌåÆ pack ŌåÆ publish **6 NuGet packages** (Contracts, Client, Server, SignalR, Testing, Aspire) + GitHub Release + CHANGELOG
- **Local run (VM/slim mode):** start `placement` + `scheduler` before `aspire run`, else actors fail with "did not find address for actor". Use `http://localhost:8080` (dev HTTPS cert not fully trusted); `EnableKeycloak=false` falls back to symmetric-key JWT

### Critical Don't-Miss Rules

- **Never** add package versions to `.csproj` ŌĆö all versions in `Directory.Packages.props`
- **Never** use `.sln`; **never** run solution-level `dotnet test`
- **Never** `Guid.TryParse` an id field ŌĆö ULIDs only
- **Never** reorder the MediatR behavior pipeline (Auth ŌåÆ Log ŌåÆ Validate)
- **Never** add copyright headers (this repo doesn't use them)
- **Never** recurse into nested submodules or modify submodule files unsolicited
- **Always** `ConfigureAwait(false)` on awaits
- **Always** assert persisted state in Tier 2/3 tests, not just status codes
- **Always** keep aggregates pure ŌĆö events in, state rebuilt, no in-place mutation
